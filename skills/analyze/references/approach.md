# Approach

Use this sequence when creating or updating a PRD:

1. Confirm the input problem, target users, and desired outcome.
2. Read repository context from the required contract files.
3. Draft from `assets/templates/prd_template.md` and keep its headings in order.
4. Convert ambiguity into explicit assumptions rather than leaving hidden gaps.
5. Keep detailed FR/AC content in `requirements.yml`, not `prd.md`.
6. Use `harness.yml` plus repository docs to decide whether feature flags, telemetry, performance requirements, issue tracking, and code review should shape the PRD.
7. Finish with checklist and validation gates before marking the PRD complete.
