# Approach

Follow this sequence to produce the FDD:

1. Study the PRD and relevant repository docs.
2. Perform lightweight codebase waypointing to find the affected boundaries and entry points.
3. Restate goals, constraints, assumptions, and success criteria in technical terms.
4. Compare a small number of design options and select the simplest adequate approach.
5. Document interfaces, data impacts, failure modes, and validation strategy.
6. Finish with checklist and validation gates.
