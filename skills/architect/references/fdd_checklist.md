# FDD Checklist

- Use the sections from `assets/templates/fdd_template.md`.
- Map design decisions back to the PRD and requirements traceability.
- Make assumptions explicit.
- Specify boundaries, interfaces, data impacts, and failure handling.
- Include observability, security/privacy, and operational considerations when relevant.
- Include a concrete testing strategy.
- Remove unresolved `TODO`, `TBD`, and `FIXME` markers.
