# Parent Context

When a small work item belongs to a larger initiative, use local docs for alignment.

## Context Inputs
- parent or epic identifier from the issue tracker when present
- local work-item docs under `docs/exec-plans/current/` or repository design docs when they exist

## Usage Rule
- parent context constrains implementation decisions and out-of-scope boundaries
- do not edit broader planning docs in this lane unless the user explicitly asks
