# Considerations

Use these considerations while drafting the PRD:

- language clarity: write in plain language that cross-functional readers can understand
- roles and permissions: call out who can do what when behavior changes depend on role boundaries
- accessibility and internationalization expectations when the feature affects end users
- reliability, security, privacy, and operational posture when relevant
- observability and success measurement when telemetry is enabled
- rollout and migration only when feature flags or operational changes are actually in scope
- repository-specific architecture, testing, and tooling constraints from the required docs
