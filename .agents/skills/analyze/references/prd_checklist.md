# PRD Checklist

- Use the headings from `assets/templates/prd_template.md` in the same order.
- Keep the document concise, plain, and implementation-ready.
- Ensure sections 6 and 7 each contain exactly `Requirements are found in requirements.yml`.
- Ensure `<work_item_dir>/requirements.yml` exists and validates.
- Make scope boundaries, assumptions, risks, and open questions explicit.
- Only include feature-flag and rollout details when the capability is enabled and relevant.
- Remove unresolved `TODO`, `TBD`, and `FIXME` markers.
