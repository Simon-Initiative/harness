# Planning Considerations

Apply these considerations when drafting `plan.md`:

- both `prd.md` and `fdd.md` must exist before planning starts
- break work into small, reviewable tasks
- include testing and verification tasks in each phase
- distribute non-functional work through the plan rather than deferring it all to the end
- use repository-local docs to decide how tooling, review, telemetry, rollout, and issue-tracker work should appear
