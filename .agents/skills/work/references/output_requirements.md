# Output Requirements

Use a two-stage output contract.

## Stage 1: Pre-Implementation Review
- issue identifier or one-line problem statement
- brief technical approach
- brief numbered implementation plan
- assumptions, risks, and test strategy
- explicit request for approval before implementation

## Stage 2: Post-Implementation Summary
- what was implemented and key files changed
- tests and verification commands run and outcomes
- any scope adjustments from the original brief plan
- residual risks or follow-up actions
