# Drift Mapping

Use this matrix to convert implementation drift into doc updates.

## Behavior or requirements drift
- Update `prd.md` and `plan.md` when user-visible behavior or acceptance expectations changed.

## Interface drift
- Update `fdd.md` when interfaces, contracts, or integration details changed.

## Data or migration drift
- Update `fdd.md` and `plan.md` when storage or rollout steps changed.

## Execution-order drift
- Update `plan.md` when phase sequence or dependencies changed.
