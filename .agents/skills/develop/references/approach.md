# Approach

Use this implementation sequence:

1. Read the work-item docs and scope to the assigned phase.
2. Implement tests alongside behavior changes.
3. Keep a running execution record while you work.
4. Use repository-local docs for stack rules, verification commands, and testing expectations.
5. Run review after tests pass when code review is enabled.
6. Update work-item docs if implementation reality diverges from the plan.
7. Finish with postflight validation.
