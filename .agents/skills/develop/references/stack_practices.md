# Stack Practices

Read the target repository's stack guidance before coding.

- `docs/STACK.md` is the source of truth for language, framework, runtime, and architecture-specific practices.
- `docs/TESTING.md` defines test types, required coverage, and preferred test utilities.
- `docs/TOOLING.md` defines verification, formatting, linting, and build commands.
- `docs/OPERATIONS.md` defines runtime, deployment, telemetry, and rollout expectations.
