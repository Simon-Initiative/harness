# Development Checklist

- confirm the selected phase and dependencies from `plan.md`
- implement only the scoped phase
- update tests in the same change set as behavior changes
- run required verification and test commands
- run a review round when code review is enabled
- update `prd.md`, `fdd.md`, or `plan.md` when implementation reality changes
- remove unresolved `TODO`, `TBD`, and `FIXME` markers in touched files
